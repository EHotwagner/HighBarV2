# Implementation Plan: Integrate BAR Unit Types into F# Client

**Branch**: `009-client-unit-types` | **Date**: 2026-04-03 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/009-client-unit-types/spec.md`

## Summary

Integrate the BAR data library (953 typed unit definitions generated by features 006-008) into the HighBar F# client so that AI bot developers can access unit stats, costs, weapons, and capabilities directly from client code. The integration adds a project reference from `HighBar.Client.fsproj` to `BarData.fsproj`, exposes a thin `Units` module for convenient lookups and capability queries, and adds integration tests verifying data access across factions and capability categories.

## Technical Context

**Language/Version**: F# / .NET 8.0  
**Primary Dependencies**: BarData library (in-repo), Google.Protobuf 3.28 (existing client dep), xUnit 2.9.3 (tests)  
**Storage**: N/A (static in-memory data from generated F# records)  
**Testing**: xUnit via `dotnet test`  
**Target Platform**: .NET 8.0 (cross-platform)  
**Project Type**: Library (F# client library with BAR data integration)  
**Performance Goals**: N/A (static data access, no hot-path impact)  
**Constraints**: Must not affect proxy communication latency; data library is read-only static data  
**Scale/Scope**: 953 unit definitions, 6 capability categories, ~5-10 new integration tests
